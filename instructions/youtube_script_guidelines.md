# YouTube動画脚本 TTS変換ガイドライン

このガイドラインは、Claude Codeが脚本をYouTube動画向けの自然で親しみやすい発話テキストに変換する際の指針です。ElevenLabs TTSの機能を最大限に活用し、視聴者に響くナレーションを実現します。

---

## 1. 基本原則

### 1.1 話し言葉として書く

脚本は「読む」ためではなく「聴く」ために最適化します。

**避けるべき書き方:**
```
本研究では、FourCastNetを用いた気象予測手法について検討を行った。
```

**推奨する書き方:**
```
今回は、FourCastNetっていうAIを使って、天気予報をやってみました。
```

### 1.2 一文は短く

- 一文は20〜30文字程度を目安に
- 複雑な文は複数の短い文に分割
- 主語と述語を近づける

---

## 2. 間（ポーズ）のコントロール

### 2.1 句読点による自然な間

| 記号 | 効果 | 使用例 |
|------|------|--------|
| `、` | 短い息継ぎ | 「今日は、天気の話をします」 |
| `。` | 文の終わり、標準的な間 | 「これが結果です。では次に進みましょう」 |
| `…` | ためらい、余韻、思考中 | 「それは…ちょっと難しいですね」 |
| `―` または `—` | 短い間、補足の挿入 | 「これは―実は重要なポイントで―見逃せません」 |

### 2.2 SSMLブレイクタグ（v2モデル用）

より精密な間のコントロールが必要な場合：

```
「ここで重要なのは」<break time="1.0s" />「データの質です」
```

**注意:**
- Eleven v3ではSSMLブレイクタグは非対応
- 1回の生成で使いすぎると不安定になる可能性あり

### 2.3 文構造による間の演出

強調したいポイントの前後に改行を入れることで、自然な間を作れます：

```
そして、ここからが本題です。

AIが予測した結果を見てみましょう。
```

## 3. 感情・トーンの表現

### 3.1 ナラティブスタイル

感情は文脈や説明で伝えます：

```
「これ、すごくないですか？」と、思わず声が大きくなってしまうような結果が出ました。
```

### 3.2 Audio Tags（Eleven v3用）

v3モデルでは角括弧 `[]` でタグを挿入できます。**重要：以下は代表的な例であり、ElevenLabsは非常に柔軟にさまざまなタグを解釈できます。脚本の文脈に合わせて、自由にタグを創造・組み合わせてください。**

#### タグの基本原則

1. **柔軟性を活かす**: 下記の例にとらわれず、表現したい感情・状況を英語で記述すればOK
2. **組み合わせ可能**: `[hesitant][nervous]` のように複数タグを重ねられる
3. **自然言語で指定**: `[speaking slowly with growing concern]` のような説明的タグも有効
4. **文脈に応じて創造**: 必要な表現がリストになければ、自分で作成してよい

#### 感情・ムード（Mood/Emotion）

代表例：
| タグ | 効果 |
|------|------|
| `[excited]` | 興奮、ワクワク |
| `[nervous]` | 緊張、不安 |
| `[calm]` | 落ち着いた |
| `[frustrated]` | イライラ、苛立ち |
| `[sorrowful]` | 悲しみ、哀愁 |
| `[curious]` | 好奇心 |
| `[surprised]` | 驚き |
| `[nostalgic]` | 懐かしさ |
| `[confident]` | 自信に満ちた |
| `[hopeful]` | 希望に満ちた |
| `[melancholic]` | 物悲しい |
| `[energetic]` | 元気いっぱい |

**応用例**: `[slightly worried]`, `[growing excitement]`, `[bittersweet]`, `[cautiously optimistic]`

#### トーン・スタイル（Tone/Style）

代表例：
| タグ | 効果 |
|------|------|
| `[cheerfully]` | 明るく |
| `[deadpan]` | 無表情、淡々と |
| `[playfully]` | 遊び心を込めて |
| `[seriously]` | 真剣に |
| `[warmly]` | 温かく |
| `[flatly]` | 平坦に |
| `[dramatically]` | ドラマチックに |
| `[casually]` | カジュアルに |
| `[matter-of-factly]` | 事実を述べるように |
| `[conversational tone]` | 会話調で |

**応用例**: `[friendly tone]`, `[professional tone]`, `[intimate whisper]`, `[theatrical]`

#### リアクション・ボーカルエフェクト（Reactions/Vocal Effects）

代表例：
| タグ | 効果 |
|------|------|
| `[laughs]` | 笑い |
| `[sighs]` | ため息 |
| `[gasps]` | 息を呑む |
| `[gulps]` | ゴクリと飲み込む |
| `[whispers]` | ささやき |
| `[clears throat]` | 咳払い |
| `[soft chuckle]` | 小さく笑う |
| `[yawns]` | あくび |
| `[coughs]` | 咳 |
| `[sniffles]` | 鼻をすする |

**応用例**: `[nervous laugh]`, `[relieved sigh]`, `[sharp intake of breath]`, `[suppressed giggle]`

#### リズム・ペース（Rhythm/Pacing）

代表例：
| タグ | 効果 |
|------|------|
| `[pauses]` | 間を置く |
| `[hesitates]` | ためらう |
| `[stammers]` | どもる |
| `[rushed]` | 急いで |
| `[slowly]` | ゆっくり |
| `[drawn out]` | 引き延ばして |
| `[dramatic pause]` | 劇的な間 |
| `[building intensity]` | 徐々に強く |

**応用例**: `[speaking faster with excitement]`, `[trailing off]`, `[picking up pace]`

#### ナレーション・語り（Narrative）

代表例：
| タグ | 効果 |
|------|------|
| `[voice-over style]` | ナレーション調 |
| `[inner monologue]` | 内なる声 |
| `[storytelling mode]` | 物語を語るように |
| `[announcing]` | アナウンス調 |
| `[documentary style]` | ドキュメンタリー風 |
| `[tutorial tone]` | チュートリアル調 |

#### ユーモア（Humor）

代表例：
| タグ | 効果 |
|------|------|
| `[sarcastic]` | 皮肉っぽく |
| `[teasing]` | からかうように |
| `[ironic]` | 皮肉を込めて |
| `[self-deprecating]` | 自虐的に |
| `[dad joke tone]` | 親父ギャグ風 |
| `[dry humor]` | 乾いたユーモア |

#### 内省的（Introspective）

代表例：
| タグ | 効果 |
|------|------|
| `[vulnerable tone]` | 弱さを見せて |
| `[heartfelt]` | 心からの |
| `[reflective]` | 内省的に |
| `[contemplative]` | 熟考するように |
| `[wistful]` | もの思いにふけって |

#### 効果音（Sound Effects）

代表例：
| タグ | 効果 |
|------|------|
| `[applause]` | 拍手 |
| `[door creaking]` | ドアのきしみ |
| `[thunder]` | 雷 |
| `[typing sounds]` | タイピング音 |

**注意**: 効果音タグは状況により効果が不安定な場合があります。

#### 使用例

**基本的な使用:**
```
[excited] みなさん、これ見てください！

[curious] でも、なぜこうなるんでしょうか…

[laughs] 実は私も最初は信じられませんでした。
```

**タグの組み合わせ:**
```
[hesitant][nervous] えーと…これ、本当に大丈夫なんでしょうか。

[whispering][conspiratorial] 実は、ここだけの話なんですが…

[excited][rushed] すごいすごい！見てください、ここ！
```

**説明的なタグ:**
```
[speaking slowly with emphasis] これが、最も重要なポイントです。

[growing excitement] そして結果は…なんと…予想を超えてきました！

[as if sharing a secret] 実はね、この方法、あまり知られてないんです。
```

**文脈に応じた創造的なタグ:**
```
[like explaining to a friend] 簡単に言うとね、こういうことなんです。

[building suspense] さて、結果は…どうなったでしょうか…

[relieved laugh] ふう、やっと動きました。
```

#### タグ活用のベストプラクティス

1. **過剰使用を避ける**: すべての文にタグを付ける必要はない
2. **重要な箇所に集中**: 感情の転換点や強調したい部分で使用
3. **自然さを優先**: タグによる演出より、テキスト自体の自然さが重要
4. **テスト必須**: 新しいタグは必ず音声生成してテストする
5. **組み合わせは2-3個まで**: タグを重ねすぎると予測不能になる

### 3.3 大文字による強調

重要な単語は大文字化で強調できます：

```
これはVERY importantなポイントです。
```

日本語の場合は、カタカナや記号で強調：

```
これは「とても」重要なポイントです！
```

## 4. YouTube動画向けの親しみやすさ

### 4.1 視聴者への呼びかけ

```
みなさん、こんにちは！
今回の動画では…

ここまで見てくれた方、ありがとうございます！
```

### 4.2 口語表現の活用

| 硬い表現 | 親しみやすい表現 |
|----------|------------------|
| ～である | ～なんですね |
| ～と考えられる | ～かもしれません |
| ～を実施した | ～をやってみました |
| 興味深い結果 | 面白い結果 |
| 検討する | 考えてみる |

### 4.3 聞き手を巻き込む

```
ここで質問です。みなさんはどう思いますか？

…正解は、こちらでした！
```

### 4.4 リアクションの挿入

```
結果を見てみると…おお、これはすごい！

予想と違いましたね。ちょっと意外でした。
```

## 5. 専門用語・数値の扱い

### 5.1 専門用語は補足説明を

```
FourCastNet、つまりAI気象予測モデルを使います。

解像度は0.25度、これはだいたい25キロメートル四方です。
```

### 5.2 数値の読み上げ

TTSが正しく読めるように変換します：

| 元の表記 | TTS用表記 |
|----------|-----------|
| `0.25°` | `0.25度` または `零点二五度` |
| `10km` | `10キロメートル` または `じゅっキロ` |
| `2024/01/15` | `2024年1月15日` |
| `50%` | `50パーセント` |
| `$100` | `100ドル` |

### 5.3 略語・頭字語

```
<!-- 避ける -->
GCMを使用します。

<!-- 推奨 -->
GCM、つまり全球気候モデルを使用します。
```

## 6. 文章構成のテンプレート

### 6.1 導入部

```
[friendly tone] みなさん、こんにちは！

今回は[トピック]について解説していきます。

この動画を見ると、[得られるメリット]がわかります。

では、さっそく始めていきましょう！
```

### 6.2 セクション移行

```
はい、ここまでが[前のセクション]の話でした。

[短い間]

続いて、[次のセクション]を見ていきましょう。
```

### 6.3 締めくくり

```
今回のまとめです。

ポイントは3つありました。
1つ目は…
2つ目は…
そして3つ目は…

この動画が参考になったら、高評価とチャンネル登録をお願いします！

それでは、また次の動画でお会いしましょう。
```

## 7. 発音の制御

### 7.1 発音辞書の活用

頻出する専門用語は発音辞書で統一：

```xml
<lexeme>
  <grapheme>FourCastNet</grapheme>
  <alias>フォーキャストネット</alias>
</lexeme>
```

### 7.2 読み仮名の直接指定

発音辞書を使わない場合は、テキスト内で指定：

```
<!-- 方法1: 括弧で補足 -->
ERA5（イーラファイブ）のデータを使います。

<!-- 方法2: 読み仮名に置換 -->
イーラファイブのデータを使います。
```

## 8. 品質チェックリスト

脚本完成時に以下を確認：

- [ ] 一文が長すぎないか（30文字以内が目安）
- [ ] 専門用語に補足説明があるか
- [ ] 数値・単位は読み上げ形式になっているか
- [ ] 適切な間（ポーズ）が入っているか
- [ ] 視聴者への呼びかけがあるか
- [ ] 口語調で書かれているか
- [ ] Audio Tagsは適切に使用されているか（v3の場合）
- [ ] 導入・締めくくりは親しみやすいか

## 9. ElevenLabs設定の推奨値

### 9.1 Stability（安定性）

| 設定 | 用途 |
|------|------|
| Creative（低め） | 感情豊かな表現、Audio Tags活用時 |
| Natural（中央） | バランスの取れた自然な発話 |
| Robust（高め） | 安定した読み上げ、ニュース調 |

### 9.2 Speed（速度）

- デフォルト: `1.0`
- 説明パート: `0.9`〜`1.0`（ゆっくりめ）
- 盛り上がり部分: `1.0`〜`1.1`（やや速め）

## 10. 避けるべきパターン

### 10.1 TTSが苦手な表現

```
<!-- 避ける -->
（笑）、www、ww
→ 発話されないか、文字通り読まれる

<!-- 代替 -->
[laughs] または ふふっ
```

### 10.2 曖昧な記号

```
<!-- 避ける -->
〜、♪、★、※

<!-- 代替 -->
明示的な言葉に置き換える
```

### 10.3 視覚的な表現

```
<!-- 避ける -->
↑この図を見てください

<!-- 代替 -->
画面に表示されている図を見てください
```
